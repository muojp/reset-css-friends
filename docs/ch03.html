第3章　reset.cssは何をリセットしているのか

3.1　html要素

sanitize.cssとress.cssでは、html要素に対しbox-sizing: border-box;を指定した上で、全称セレクタへbox-sizing: inherit;を指定しています。このことでコンテンツ領域にpaddingやborderの値が入るようになります。これによりボックスサイズの計算をより簡単にすることを狙っていると思われます。

3.2　body要素

reset.css側ではEric Meyer、Normalize.css、Sanitize.css、YUI 3ではbody要素へ対しmargin: 0;を指定しています。Webページを作るときにページの外周へmarginを設定することはほとんど無いため、このような指定がされていると予想しています。

3.3　p要素

reset.css側ではYUI 3やEric Meyerといった古めのreset.cssではmarginとpaddingが0と指定されています。しかし、他のreset.cssでは特に指定がなくユーザーエージェントスタイルシートをそのまま使うようにしています。

これは新しく縦書きという概念が出てきたのに従い、reset.css側で文字の方向や向き、流れる方向を意識するようになったためです。その結果ユーザーエージェントスタイルシートをそのまま使ったほうが良いという結論に至ったと思われます。

3.4　h1要素

normalize.cssやsanitize.cssではこの入れ子具合によってスタイルが変わるのを無くし、リスト3.1で示すスタイルが適用されるようになっています。

リスト3.1: h1要素に対するnormalize.cssなどのスタイル定義

h1 {
  font-size: 2em;
  margin: 0.67em 0;
}

また、YUI3やEric Meyerでは見出しらしいスタイル定義をすべて無かったことにしています（リスト3.2）。

リスト3.2: h1要素に対するYUI3などのスタイル定義

h1 {
  margin:0;
  padding:0;
  font-size:100%;
  font-weight:normal;
}

3.5　a要素

a要素は各ブラウザのユーザーエージェントスタイルシートでは特にスタイルが適用されていません。ただ、normalize.cssやsanitize.css、ressではリスト3.3のようなスタイル定義をしています。

リスト3.3: a要素に対するreset.css側のスタイル定義

a {
  background-color: transparent;
  -webkit-text-decoration-skip: objects;
}

background-color: transparent;が指定されている理由としては、IE 10上でリンクをクリックしたときにグレーの背景がついてしまうのを無くすために定義されています。また-webkit-text-decoration-skip: objects;という指定は、英語のpやy、一部のロシア語をリンクの文字にした場合、リンクの下線が途切れてしまうのを防ぐためです*1。

[*1] https://github.com/necolas/normalize.css/pull/573

3.6　img要素

normalize.cssやsanitize.css、ressではIE10でリンク内に画像があるとborderが適用されてしまうのを防ぐために、border-style: none;が指定されています。
